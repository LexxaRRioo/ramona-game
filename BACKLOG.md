## Bugs (known, not yet fixed)

- [x] Lie-down/curl-into-sleep transition should pick one side (left-curl or right-curl) at random each time and stick to frames from only that side - row 6's original 10-frame sequence mixed both curl directions. Split into `lieDownLeft`/`lieDownRight` (+ matching `sleepLeft`/`sleepRight`), one picked at random per settle - pending release.
- [x] Ramona sits in front of the Dock instead of on it - the Dock's AXList frame from Accessibility reports ~5pt more height than the strip's actual rendered top edge (confirmed via screenshot pixel-sampling), so she settled floating just above/in front of the visible glass. Fixed with a measured correction in `Dock.bottomFrame()` - pending release.

- [ ] Window/Dock floor tracking is fragile and keeps regressing - test-covered and fixed this session, but NOT yet verified live (leave unchecked until confirmed - see last bullet below):
  - Root cause 1 (window tracking): `FrontmostWindowTracker` only ever tracked "whichever window is currently frontmost", with no concept of window identity - so if she'd climbed onto window A and the user then focused window B (without moving A), the tracker silently started reporting B's frame through the exact same callback used for "your window moved", and she'd follow it. Fixed by having the tracker compare AXUIElements across reports (`CFEqual`) and pass an `isSameWindow` flag through `onFrameChange`; `FloorTracking.nextSurface` (new, pure, unit-tested in `FloorTrackingTests`) decides the resulting surface, and `CatScene` uses `isSameWindow` to pick the animation - a same-window move re-settles normally, a different window becoming frontmost plays an explicit walk-style jump (`playJumpToCurrentSurface`) instead of silently gliding across in whatever pose she was in.
  - Root cause 2 (Dock sinking/floating): `Dock.topPaddingCorrection` was a fixed 5pt guessed from one machine's Dock. The padding above the Dock's visible icons scales with icon size (System Settings > Dock > Size), so a constant can't generalize - this dev machine's Dock (tile size 66, magnification on) is a different config than whatever 0.2.0 was tuned against. Replaced with a direct measurement: `Dock.bottomFrame()` now unions the individual icon elements' own AX frames (median of their top edges, robust to a single magnified/hovered outlier) instead of applying a constant to the outer list frame - see `Dock.strip`/`Dock.median`, unit-tested in `DockGeometryTests`.
  - Extracted `Surface`/`FloorTracking` (window > Dock > floor ground-line priority, perch validity, and the surface-transition decision) out of `CatScene` into a plain, AppKit/SpriteKit-free type specifically so this logic has real test coverage instead of only being checkable by eyeballing the running app.
  - **Not independently verified live**: pixel-level Dock alignment needs Accessibility permission granted to a running build, which isn't something achievable from this sandboxed session - build via `Scripts/build-dev-app.sh`, grant Accessibility once to `.build/Ramona.app`, and confirm she now sits cleanly on the Dock strip (not floating or sinking) and correctly jumps between windows as focus changes.
